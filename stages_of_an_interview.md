Stages of an interview
1.- Introductions
2.- Problem statement
2.1.- Make sure to understand the problem, confirm what the problem is asking by paraphrasing it back to them.
2.2.- Ask clarifying questions regarding the input, for example:
-- Will the input only have integers, or could there be other types?
-- Will the input be sorted or unsorted?
-- Is the input guaranteed to have elements or could it be empty?
-- What should I do if an invalid input is given?
2.3.- Ask about the expected input size
3.- Brainstorming DS&A
3.1.- Try to figure out what data structure or algorithm is applicable. 
Think out loud. It will show your interviewer that you are good at considering tradeoffs.
4.- Implementation
5.- Testing & debugging
6.- Explanations and follow-ups
7.- Outro
